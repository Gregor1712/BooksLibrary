Úvod
Tento dokument je sprievodca štúdiom pre nového .NET + Angular developera, ktorý nastupuje na
projekt Masha. Predchádzajúca skúsenosť vývojára: Python (interné aplikácie v Cenekone). Cieľom je
dať mu prehľad o technológiách, s ktorými sa stretne, a o verejných materiáloch, na ktorých sa môže
pripraviť pred podpisom zmluvy.
Dôležitý kontext k prístupom:
Prístup k repu Masha, internému kódu, firemnému emailu a interným nuget feedom
(nuget.kros.wtf, Azure DevOps krossk/Masha) dostane až po podpise zmluvy a pridelení
firemného emailu.
Pred nástupom má k dispozícii iba verejné zdroje — všetky materiály v tomto dokumente sú voľne
dostupné.
Rýchlosť štúdia je individuálna, dokument neurčuje konkrétny rozvrh.

Vývojár sa bude aktívne podieľať na dvoch paralelných upgrade projektoch.

Vrstva Z čoho Na čo

breaking changes. .NET 9 sa preskakuje (je STS, nie LTS).

Vrstva Z čoho Na čo

Praktický cvičný projekt (odporúčaný)
Najefektívnejší spôsob štúdia nie je čítanie dokumentácie, ale postaviť si vlastný mini-projekt na
rovnakom stacku ako Masha. Po skončení bude vedieť, čo ho v reálnom projekte čaká.
Backend — vlastné API (napr. “Library API”)
1. Klonovať Kros.AspNetCore.BestPractices, rozbehnúť cez docker-compose (Redis + SQL Server).
2. Postaviť novú vlastnú službu (knihy/výpožičky/používatelia, alebo vlastná téma):
.NET 10 (nie .NET 8 — cieľová verzia po upgrade Masha)
ASP.NET Core Web API
Kros.KORM (verejný NuGet)
MediatR pre CQRS
FluentValidation pre validáciu
Mapster pre mapping
xUnit + NSubstitute pre testy
3. Pridať Azure Function (timer trigger) — napr. denný export do CSV.
Frontend — Angular SPA napojená na backend
1. 
ng new mini-projekt --version=15 — cieľavedome stará verzia (rovnaká ako v Mashe), aby si zažil
východiskový stav.
2. PrimeNG 15 + NgRx 15 (Store + Effects + Entity).
3. ngrx-forms pre form state.
4. Jednoduchý CRUD UI napojený na backend z prvej časti.
Skúsiť upgrade na vlastnom projekte
1. Postupne 
ng update na novšie verzie Angularu (16 → 17 → 18 → …).
2. Sledovať breaking changes na každom kroku.
3. Prerobiť na standalone components, nový control flow, signals.
4. Vyskúšať PrimeNG migráciu (15 → vyššia, nový Aura theming).
5. Paralelne urobiť aj .NET upgrade v backend projekte (8 → 10), riešiť breaking changes Kros.KORM,
MediatR a iných knižníc.
Takto si vývojár zažije oba upgrade procesy v malom merítku ešte pred tým, ako sa s nimi stretne v
Mashe.

Odporúčania pre vývojára
1. Nainštalovať si Claude Code (alebo aspoň vyskúšať claude.ai cez web) ešte pred nástupom. Pri
cvičnom projekte (mini “Library API” + Angular SPA) ho používať aktívne — generovanie
controllerov, NgRx slice-ov, testov.
2. Naučiť sa brainstorming workflow — pred akoukoľvek väčšou zmenou si s AI premyslieť zámer a
dizajn. Ušetrí veľa zlej cesty.
3. AI nie je auto-pilot — generovaný kód treba vždy prečítať, pochopiť a otestovať. Zodpovednosť
za zmergený kód je vždy na vývojárovi.
4. Pri upgrade projektoch je AI veľmi silný spojenec — vie spoľahlivo prerobiť stovky súborov na
nový control flow alebo standalone components, čo by manuálne trvalo dni.
Po nástupe (až mu pridelíme prístupy)
Tieto kroky nemá zmysel pripravovať pred zmluvou, ale je dobré, aby vedel, čo ho čaká:
1. Onboarding na KROS prostredie — Azure DevOps, dev VM, prístup do interných NuGet feedov
(nuget.kros.wtf, pkgs.dev.azure.com/krossk), @kros-sk/* npm balíky.
2. Mentor mu prejde štruktúru Masha v páre — orientácia v repozitári, build a deploy flow.
3. Prvý jednoduchý ticket — typu “oprav text v UI” alebo “pridaj jedno pole do existujúceho
endpointu”. Zažije celý cyklus PR → review → merge → deploy.
4. Postupne väčšie úlohy a participácia na upgrade projektoch (najprv .NET, potom Angular).
Odporúčané poradie samotných upgradeov
Pre úspech projektu je rozumné:
1. Najprv .NET upgrade (8 → 10) — rýchla výhra, naučí workflow s upgrade PR a regresnými testami.
Typicky úprava 
TargetFramework + oprava niekoľkých breaking changes v Kros.* knižniciach.
2. Potom Angular upgrade (15 → 20/21) — dlhší a náročnejší. Šesť major krokov, každý s vlastným
hlavolamom (PrimeNG, ngrx-forms, build system, control flow, signals).
